# Contributing to Qualys-ToolBox

Thank you for considering contributing to Qualys-ToolBox! We welcome 
contributions from the community to help make this project better. 
Before you get started, please take a moment to review the following 
guidelines.

## Code of Conduct

Please read and adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).
We expect all contributors to follow it to ensure a welcoming and inclusive 
environment for everyone.

## How to Contribute

Here are the general steps to contribute to Qualys-ToolBox:

1. **Fork the Repository**: Click the "Fork" button on the top right corner of 
this repository to create a copy in your GitHub account.
2. **Clone the Repository**: Clone your forked repository to your local machine 
using the following command:
```shell
git clone https://github.com/yourusername/qualys-toolbox.git 
```
3. **Create a Branch**: Create a new branch for your contribution:
```shell
git checkout -b feature-or-fix-branch
```
4. **Make Changes**: Make your code changes, add new features, or fix bugs. Be sure
to follow the coding style and conventions of the project.
5. **Commit Changes**: Commit your changes with a descriptive commit message:
```shell
git commit -m "Your commit message here"
```
6. **Push Changes**: Push your branch to your GitHub fork:
```shell
git push origin feature-or-fix-branch
```
7. **Open a Pull Request**: Go to the [Pull Requests](https://github.com/derrynEdwards/qualys-toolbox/pulls)
section of the original repository and click the "New Pull Request" button. Provide a clear title
and description for your pull request.
8. **Review and Collaboration**: Your pull request will be reviewed by project maintainers. Be
prepared to address any feedback or make necessary changes.
9. **Merge**: Once your pull request is approved. It will be merged into the main project.

## Development Setup

If you're planning to contribute regularly, you may want to set up a development environment.

## Reporting Issues

If you encounter bugs or have feature requests, please open an issue on the [issues](https://github.com/derrynEdwards/qualys-toolbox/issues)
page. Provide detailed information about the problem or the new feature you'd like to see.

## Thank You

Your contributions help make Qualys-ToolBox better for everyone. We appreciate your time and
effort in contributing to this project!
